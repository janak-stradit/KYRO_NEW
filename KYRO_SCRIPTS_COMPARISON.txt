2. SCRIPT CATEGORIES:

3. DYNAMIC MESSAGE GENERATION:
   - Functions that take parameters and return formatted text
   - Example: monitoringGoals(goalCount: number)
   - Contextual messaging based on actual data

4. RICH CONTEXT FORMATTING:
   - formatAutonomousCaseContext() - Case summaries with triggers, tiers, scores
   - extractDrivers() - Identifies risk drivers from case data
   - compactCaseRef() - Short case references (customer • case-id)
   - formatCaseList() - Shows cases with "+N more" overflow

5. RISK NARRATION SYSTEM:
   - Explains why each case was flagged
   - Shows score transparency (static/behavioral/composite)
   - Tracks tier changes (LOW → MEDIUM → HIGH)
   - Describes audit trail

6. ACTION OUTCOME NARRATION:
   - Success/failure messages for each action type
   - Detailed results (e.g., "5 resolved, 3 escalated, 2 re-queued")
   - Case-by-case breakdown with triggers and tiers

7. TOOL OUTCOME NARRATION:
   - Standardized responses for 11+ tools
   - search_cases, get_case_details, trigger_review, etc.
   - Handles not_found, ambiguous, empty states

8. TYPESCRIPT TYPE SAFETY:
   - All functions are typed
   - Outcome type: "SUCCESS" | "FAILURE" | "PARTIAL" | "PENDING"
   - Interface definitions for data structures

EXAMPLE:
```javascript
function formatCaseList(cases, maxShow = 5) {
    if (!cases || cases.length === 0) return "none";
    const shown = cases.slice(0, maxShow);
    const remaining = cases.length - shown.length;
    const list = shown.map(c => `${c.customer_id} • ${c.case_id.slice(0,8)}`).join(", ");
    return remaining > 0 ? `${list} +${remaining} more` : list;
}
```
