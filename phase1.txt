# Phase 1: Foundation & Data Infrastructure

## Overview
Build the core backend, database, data pipeline, and basic API for the KYRO Risk Assessment Specialist system.

**Duration:** 3–4 weeks
**Goal:** Core backend, database, data pipeline, and basic API

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend Framework | Python 3.11+, FastAPI |
| Database | PostgreSQL 15+ |
| Cache / Session | Redis |
| Task Queue | Celery + Redis |
| Containerization | Docker, Docker Compose |
| Testing | pytest, httpx |

## Database Schema

### 1. Customers Table
```sql
CREATE TABLE customers (
    customer_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    full_name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    phone VARCHAR(50),
    date_of_birth DATE,
    country VARCHAR(100),
    residency_country VARCHAR(100),
    kyc_status VARCHAR(50) DEFAULT 'PENDING', -- PENDING, VERIFIED, REJECTED, UNDER_REVIEW
    kyc_last_review TIMESTAMP,
    pep_flag BOOLEAN DEFAULT FALSE,
    sanctions_flag BOOLEAN DEFAULT FALSE,
    adverse_media_flag BOOLEAN DEFAULT FALSE,
    risk_level VARCHAR(20) DEFAULT 'LOW', -- LOW, MEDIUM, HIGH
    risk_score INTEGER DEFAULT 0 CHECK (risk_score >= 0 AND risk_score <= 100),
    customer_type VARCHAR(50), -- INDIVIDUAL, CORPORATE, FUND
    customer_metadata JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2. Customer Risk Profiles Table
```sql
CREATE TABLE customer_risk_profiles (
    profile_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    risk_category VARCHAR(50), -- GEOGRAPHIC, PRODUCT, CHANNEL, BEHAVIORAL
    risk_factor VARCHAR(255),
    risk_weight DECIMAL(5,2),
    assessed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    assessed_by VARCHAR(50) DEFAULT 'SYSTEM'
);
```

### 3. KYC Reviews Table
```sql
CREATE TABLE kyc_reviews (
    review_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    review_type VARCHAR(50), -- PERIODIC, TRIGGERED, ADHOC
    review_status VARCHAR(50) DEFAULT 'SCHEDULED', -- SCHEDULED, IN_PROGRESS, COMPLETED, OVERDUE
    scheduled_date DATE,
    completed_date TIMESTAMP,
    findings TEXT,
    risk_level_after VARCHAR(20),
    reviewed_by UUID,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. PEP / Sanctions Screening Table
```sql
CREATE TABLE pep_sanctions_screening (
    screening_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    screening_type VARCHAR(50), -- PEP, SANCTIONS, ADVERSE_MEDIA
    match_status VARCHAR(50), -- NO_MATCH, POTENTIAL_MATCH, CONFIRMED_MATCH
    match_details JSONB,
    screened_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    screened_by VARCHAR(50) DEFAULT 'SYSTEM',
    resolution VARCHAR(50), -- CLEARED, ESCALATED, CONFIRMED
    resolved_at TIMESTAMP,
    resolved_by UUID
);
```

### 5. Accounts Table
```sql
CREATE TABLE accounts (
    account_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    account_type VARCHAR(50), -- CHECKING, SAVINGS, INVESTMENT, TRADING
    account_status VARCHAR(50) DEFAULT 'ACTIVE', -- ACTIVE, SUSPENDED, CLOSED, FROZEN
    currency VARCHAR(3) DEFAULT 'USD',
    balance DECIMAL(18,2) DEFAULT 0.00,
    opened_date DATE,
    account_metadata JSONB,
    risk_flags JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6. Account Metadata Table
```sql
CREATE TABLE account_metadata (
    metadata_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id UUID REFERENCES accounts(account_id) ON DELETE CASCADE,
    meta_key VARCHAR(100),
    meta_value TEXT,
    meta_type VARCHAR(50), -- STRING, NUMBER, BOOLEAN, JSON
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 7. Account Balances Table
```sql
CREATE TABLE account_balances (
    balance_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id UUID REFERENCES accounts(account_id) ON DELETE CASCADE,
    balance DECIMAL(18,2),
    available_balance DECIMAL(18,2),
    currency VARCHAR(3),
    recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 8. Transactions Table
```sql
CREATE TABLE transactions (
    transaction_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    account_id UUID REFERENCES accounts(account_id) ON DELETE CASCADE,
    transaction_date TIMESTAMP NOT NULL,
    transaction_type VARCHAR(50), -- DEPOSIT, WITHDRAWAL, TRANSFER, FX, TRADE
    amount DECIMAL(18,2) NOT NULL,
    currency VARCHAR(3) DEFAULT 'USD',
    meta_counterparty VARCHAR(255),
    meta_counterparty_type VARCHAR(50), -- INDIVIDUAL, CORPORATE, FINANCIAL_INSTITUTION
    meta_location VARCHAR(255),
    meta_country VARCHAR(100),
    meta_country_code VARCHAR(3),
    meta_destination_country VARCHAR(100),
    meta_origin_country VARCHAR(100),
    meta_source VARCHAR(100), -- API, BATCH, MANUAL, SWIFT
    risk_flags JSONB,
    risk_score INTEGER DEFAULT 0,
    source_system VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 9. Transaction Counterparties Table
```sql
CREATE TABLE transaction_counterparties (
    counterparty_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    transaction_id UUID REFERENCES transactions(transaction_id) ON DELETE CASCADE,
    counterparty_name VARCHAR(255),
    counterparty_type VARCHAR(50),
    counterparty_country VARCHAR(100),
    counterparty_account VARCHAR(100),
    bank_name VARCHAR(255),
    bank_country VARCHAR(100),
    relationship_to_customer VARCHAR(100)
);
```

### 10. Transaction Risk Flags Table
```sql
CREATE TABLE transaction_risk_flags (
    flag_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    transaction_id UUID REFERENCES transactions(transaction_id) ON DELETE CASCADE,
    flag_type VARCHAR(100), -- AMOUNT_THRESHOLD, VELOCITY, GEOGRAPHY, COUNTERPARTY, BEHAVIORAL
    flag_description TEXT,
    flag_severity VARCHAR(20), -- LOW, MEDIUM, HIGH, CRITICAL
    triggered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    triggered_by VARCHAR(50) DEFAULT 'RULES_ENGINE'
);
```

### 11. Alerts Table
```sql
CREATE TABLE alerts (
    alert_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(customer_id) ON DELETE CASCADE,
    alert_type VARCHAR(100), -- BEHAVIORAL_ANOMALY, VELOCITY, GEOGRAPHY, PEP, SANCTIONS
    risk_score INTEGER NOT NULL,
    confidence DECIMAL(5,2),
    triggered_rules JSONB,
    ml_explanation JSONB,
    recommended_action VARCHAR(100), -- REVIEW, ENHANCED_DUE_DILIGENCE, SAR, CLOSE
    status VARCHAR(50) DEFAULT 'OPEN', -- OPEN, ASSIGNED, IN_REVIEW, RESOLVED, ESCALATED
    assigned_to UUID,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    resolved_at TIMESTAMP,
    resolved_by UUID,
    resolution_notes TEXT
);
```

### 12. Audit Logs Table
```sql
CREATE TABLE audit_logs (
    log_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    entity_type VARCHAR(50), -- CUSTOMER, ACCOUNT, TRANSACTION, ALERT
    entity_id UUID,
    action VARCHAR(100), -- CREATE, UPDATE, DELETE, REVIEW, APPROVE, REJECT
    performed_by UUID,
    performed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    old_values JSONB,
    new_values JSONB,
    ip_address INET,
    user_agent TEXT
);
```

### 13. Users Table (for RBAC)
```sql
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    full_name VARCHAR(255),
    role VARCHAR(50) DEFAULT 'ANALYST', -- ANALYST, COMPLIANCE_OFFICER, ADMIN
    is_active BOOLEAN DEFAULT TRUE,
    last_login TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Data Ingestion Pipeline (KYRO Module)

### ETL Flow
```
Source System → Extract → Validate → Transform → Load → PostgreSQL
```

### Components
1. **Extractor:** Pull data from source_system via API, SFTP, or database connection
2. **Validator:** Schema validation, data quality checks, deduplication
3. **Transformer:** Normalize formats, enrich with reference data, calculate derived fields
4. **Loader:** Bulk insert into PostgreSQL with transaction safety

## Rules Engine

### Implemented Rules
| Rule ID | Name | Description | Severity |
|---------|------|-------------|----------|
| R001 | Amount Threshold | Transaction > $10,000 | MEDIUM |
| R002 | Velocity Daily | > 5 transactions/day | MEDIUM |
| R003 | Velocity Hourly | > 3 transactions/hour | LOW |
| R004 | High Risk Country | Counterparty in sanctioned/high-risk country | HIGH |
| R005 | PEP Match | Customer or counterparty is PEP | HIGH |
| R006 | Sanctions Match | Customer or counterparty on sanctions list | CRITICAL |
| R007 | New Counterparty | First-time counterparty | LOW |
| R008 | Weekend Activity | Unusual weekend transaction pattern | LOW |
| R009 | Round Amount | Suspicious round amount (e.g., $10,000.00) | MEDIUM |
| R010 | Rapid Succession | Multiple transactions within 1 minute | HIGH |
